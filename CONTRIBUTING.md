# Contributing

Thanks for contributing to Roc.

This guide covers the human workflow for editing icons, docs, and generated outputs. For the detailed icon drawing rules, see [CLAUDE.md](CLAUDE.md).

## Local Setup

Roc uses a single Node-based build script and publishes generated files from `dist/`.

```bash
npm install
npm run build
npm run preview
```

Notes:

- `npm install` runs `prepare`, which builds `dist/` locally.
- `dist/` is generated and gitignored.
- `demo/index.html` is generated by the build and committed to the repo.

## Repository Map

| Path | Purpose |
| --- | --- |
| `src/svg/{style}/` | Source-of-truth SVG files, one icon variant per file |
| `src/icons.json` | Labels, descriptions, categories, and tags |
| `build.mjs` | Full build pipeline for SVG, React, Svelte, sprite, metadata, and demo output |
| `demo/src/` | Source files for the generated demo page |
| `docs/api.md` | Public package API reference |
| `README.md` | External quick-start and usage guide |
| `docs/spec.md` | Internal architecture notes |
| `CLAUDE.md` | Detailed icon design rules and naming conventions |

## Adding or Updating an Icon

1. Create or update all four SVG variants in `src/svg/{outline,solid,duotone,sharp}/`.
2. Add or update the metadata entry in `src/icons.json`.
3. Run `npm run build`.
4. Open the demo with `npm run preview` and inspect the new icon in every style.
5. Run `npm run check:types` before opening a PR.

`CLAUDE.md` is the authoritative guide for:

- root SVG requirements
- per-style geometry and attribute rules
- naming conventions
- example source SVGs

## Generated Files

Generated outputs are expected to be reproducible from source:

- `dist/` is the package build output and should be regenerated locally, not edited by hand.
- `demo/index.html` is generated from `demo/src/*` by `npm run build`; do not edit it directly.
- If you change demo source files or icon data, include the regenerated `demo/index.html` in your PR.

The current build overwrites generated files but does not prune stale files from `dist/`. If you rename or remove icons locally, delete `dist/` before rebuilding so you are testing a clean output tree.

## Verification

Run these commands before opening a PR:

```bash
npm run build
npm run check:types
```

Use `npm run preview` for a manual pass on the demo UI when your change affects icons, metadata, or demo rendering.

`npm run verify:package` currently runs the same Svelte package type check as `npm run check:types`.

## Pull Requests

Keep PRs focused and explain what changed at the package surface:

- what icons, docs, or build behavior changed
- what verification you ran
- whether `demo/index.html` was regenerated
- any intentional follow-up work left out of scope

If your PR changes icon design rules or contributor workflow, update the relevant docs in the same branch so `README.md`, `docs/api.md`, `CONTRIBUTING.md`, `CLAUDE.md`, and `docs/spec.md` stay aligned.
